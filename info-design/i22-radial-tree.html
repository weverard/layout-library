Spec — what it is, when to use it
What it is
A radial tree: a root at the center, branches at a fixed inner radius, and leaves at a fixed outer radius, joined by curved links that bow outward along the radius rather than running as straight spokes.
When to use it
Org charts, topic taxonomies, or any two-level hierarchy where one branch is the story and the rest are context — the circular form gives every branch room without a branch competing for the same vertical space.
Data shape
{label, depth, angle, r, parent}[] — a root, 6 branches, 2-4 leaves per branch; every angle and radius precomputed so render code only draws, never lays out.
Visual style
One branch and its leaves carry the accent; the rest stay a neutral ink-gray. Labels rotate to match their own node's angle and flip upright on the left half of the circle so nothing reads upside down.

This is a radial tree: a small root dot sits at the center of a circle, six branch dots ring it at a fixed inner radius, and their leaf dots sit further out at a fixed outer radius. Curved links join parent to child, each one bowing out along the radius instead of running as a straight spoke, so the shape reads as organic rather than mechanical. One branch and every leaf beneath it carry the accent color; the rest of the tree stays a neutral ink-gray, so the eye finds the story branch immediately. Every label is rotated to match its own node's angle around the circle and flips upright on the left half so nothing reads upside down, the same trick used for the ring labels in the chord diagram. Clicking any branch dot collapses its leaves — the links and leaf dots fade out in place — and clicking again brings them back, so a dense tree can be explored one branch at a time. Under roughly 560px the circular layout gives way entirely to an indented outline list built from the exact same parent-linked data, with the same click-to-collapse behavior on each branch row.
