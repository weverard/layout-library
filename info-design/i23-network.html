Spec — what it is, when to use it
What it is
A node-link network graph: discs sized by weight, joined by low-opacity links, laid out so the underlying structure reads as two separable clusters.
When to use it
Collaboration graphs, service dependencies, influence maps — anywhere the question is "who connects to whom" and whether any single node is a structural risk.
Data shape
nodes[]{id,label,x,y,w}, links[]{a,b} — 14 nodes, 18 links; positions and radii are precomputed, not laid out by a physics simulation at render time.
Visual style
Disc radius is proportional to the square root of weight so area (not radius) encodes it; only the six heaviest nodes carry a permanent label; the sole node bridging both clusters carries the accent and a small "the broker" note.

This is a node-link network graph: fourteen discs sized by the square root of a weight value, so their area rather than their radius encodes it, joined by thin ink-colored links held at low opacity so the overall density reads as texture rather than clutter. The positions are precomputed rather than force-simulated, arranged so the graph visibly splits into two separate clusters with open space between them. Only the six heaviest nodes carry a permanent label; the rest are identifiable only on hover, keeping the resting state calm. One node sits in the gap between the clusters and is the only path between them — it carries the accent color and a small italic note naming it the broker, so the structural risk of a single point of failure is visible without reading the data. Hovering any node holds it and its direct neighbors at full opacity and fades every other node and link, so a reader can trace one person's whole set of connections at a glance. Under roughly 560px the labels thin further, down to just the top three heaviest nodes, so names don't collide at small sizes.
