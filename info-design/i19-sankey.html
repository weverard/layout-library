Spec — what it is, when to use it
What it is
A Sankey diagram: node bars at two columns, joined by ribbons whose width is proportional to the flow between them.
When to use it
Anywhere volume moves from a set of sources to a set of outcomes and both the total at each node and the individual paths between them matter — traffic sources into conversion states, budget into line items, energy into end uses.
Data shape
nodes[]{label,x,y0,y1}, links[]{s,t,v,sy0,sy1,ty0,ty1} — 3 sources, 4 targets, 10 links; every node's height and every ribbon's attachment band is precomputed from the flow values times a shared scale.
Visual style
Solid dark node bars anchor the columns; ribbons sit at low opacity so crossings stay legible, with only the single largest link carrying the accent color. Labels sit outside the bars; exact values surface on hover.

This is a Sankey diagram: two columns of solid dark bars — three source nodes on the left, four target nodes on the right — connected by soft cubic-bezier ribbons whose thickness is proportional to the flow they carry. Every ribbon is drawn at low opacity in a neutral warm gray so overlapping paths stay readable, except the single largest flow in the whole diagram, which is the only ribbon carrying the accent color, so the eye lands on the dominant path first. Node labels sit outside the bars — left-aligned to the right of source bars, right-aligned to the left of target bars — rather than inside them, keeping the bars themselves clean. There's no value axis: hovering a ribbon isolates it, fading every other ribbon and popping a small tooltip with the exact flow value, and hovering a node shows its total. Under roughly 560px the whole diagram re-renders vertically — sources stacked on top, targets stacked on the bottom, ribbons flowing top to bottom instead of left to right — recomputed from the same underlying numbers rather than swapped for a different chart.
