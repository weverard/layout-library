Spec — what it is, when to use it
What it is
A row of thin stems ending in dots, one per item, crossed by a single labeled benchmark line; dots are colored by which side of the benchmark they land on.
When to use it
Any ranking that's really a pass/fail question against one reference value — a target, an SLA, a peer median — where the count that clears the bar matters as much as the order.
Data shape
{label, v}[] (12 items) plus a single BENCHMARK value; author the sort order — descending reads best.
Visual style
Neutral ink stems and dots for items short of the benchmark, accent for items past it; a dashed vertical rule marks the benchmark with its value labeled at the top.

Recreate prompt
This is a lollipop chart: twelve thin horizontal stems, each ending in a small dot, stacked in rows and already sorted from highest value to lowest. A single dashed vertical rule cuts across every row at one fixed x-position — the benchmark — with its value set in small mono type just above where the line begins. Stems and dots that fall short of the benchmark stay a plain ink color; the moment a dot's value passes the benchmark, both its stem and dot switch to the accent color, so the split reads instantly without needing to check the axis. There's no value axis running along the bottom — the benchmark line and the two-tone dots carry all the comparison a reader needs. Hovering a dot raises a small dark tooltip with its exact value. On narrow screens the stems simply get thinner rather than the layout changing, since the whole chart scales down as one vector rather than reflowing.
