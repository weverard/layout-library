Spec — what it is, when to use it
What it is
A proportional-area comparison: marks are scaled so radius/side ∝ √v, which makes AREA proportional to value — the honest way to size a shape by magnitude.
When to use it
Magnitude gaps too wide for bars to hold gracefully on one axis ("A is 40x B") — a bar that tall or that short reads worse than a shape that big or that small.
Data shape
{label, v}[], 4 items, plus a shape flag (circle | square) so a forker can swap the mark without touching the scaling math.
Visual style
Shapes sit on a common baseline (not center-aligned) so the ground plane reads like a real comparison; the largest is called out in accent; value and label sit directly beneath each shape; an honesty footnote states plainly that area, not radius, carries the value.

This is a proportional-area shape comparison: four marks — circles by default, swappable to squares — are sized so their linear dimension is proportional to the square root of the value, which makes their AREA proportional to the value itself, the honest way to encode magnitude in a filled shape. The shapes sit on a shared baseline rather than being center-aligned, the way objects would actually rest on a ground line, with a value and a short label set directly beneath each one instead of a legend. The largest shape carries the accent color; the rest stay a quiet neutral, so the eye lands on the biggest number first. A small italic footnote states the honesty rule outright — "Area (not radius) is proportional to value" — because area comparisons are easy to misjudge and the chart should not rely on the reader's intuition alone. Hovering a shape raises a chip with its exact value. Under about 520px the row of shapes wraps into a 2-by-2 grid instead of squeezing four abreast.
