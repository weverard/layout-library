Spec — what it is, when to use it
What it is
A beeswarm / dot strip — every individual value plotted as its own dot along a shared value axis, jittered vertically within its category's row so overlapping points stay visible instead of stacking exactly on top of each other.
When to use it
Small-n distributions where each point is a real, nameable thing — deals, tickets, applicants — and collapsing them into a bar or box would hide the individuals that matter.
Data shape
{cat, v, dy}[] — 3 categories x ~40 values (120 points); the vertical jitter (dy) is hardcoded per point, never randomized at runtime. A separate small OUTLIER constant names the one point that gets a leader-line callout.
Visual style
One category in --accent, the other two in neutral gray; a heavy tick plus label marks each row's median; the named outlier gets a thin leader line to a short caption instead of being left for the reader to spot.

This is a beeswarm (dot strip): three horizontal bands, one per category, each holding roughly forty small dots plotted at their true value along one shared value axis and jittered vertically so that nearby points fan out instead of overlapping into an unreadable smear. One category's dots are drawn in the accent color; the other two stay a neutral gray, so a single glance shows which group the story is about. A heavy tick crosses each band at its median with a small value label, giving every row a fixed reference point the individual dots scatter around. One dot — a genuine outlier, far past where the rest of its category clusters — carries a thin leader line out to a short caption naming it directly, since a chart of ~120 individual points is exactly the place where the one unusual value deserves its own sentence rather than being folded into an average. Hovering any dot reveals a small chip with its exact value. Below roughly 520px the rows grow taller and the vertical reference gridlines thin out to keep the now-larger dots legible at a small size.
